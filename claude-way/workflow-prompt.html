You are helping a solopreneur build a master routing file for their AI skill system. They have three skill files: worldview.md (philosophy), brand.md (identity), and execution.md (methodology). Generate a system.md routing OS and routing annotations for each file. The routing logic must be built from their actual workflow — not a generic template. Reference their specific task types, failure patterns, quality gate, and ideal session. --- MY WORKFLOW ANSWERS: Tasks I use AI for most: [LIST YOUR MAIN AI TASKS] The task where getting it wrong costs most: [YOUR ANSWER] What a typical session looks like: [YOUR ANSWER] When AI disappoints me, it's usually because: [YOUR ANSWER] The one thing AI always needs to know: [YOUR ANSWER] What I check before publishing anything AI helped with: [YOUR ANSWER] When I override AI and write it myself: [YOUR ANSWER] What a perfect AI session looks like: [YOUR ANSWER] --- MY SKILL FILES: WORLDVIEW.MD: [PASTE WORLDVIEW.MD CONTENT HERE] BRAND.MD: [PASTE BRAND.MD CONTENT HERE] EXECUTION.MD: [PASTE EXECUTION.MD CONTENT HERE] --- Generate two outputs: 1. A complete system.md file with: YAML frontmatter (name and description), How to Load This System section, Task Routing Rules (one per task type listed — each with: load first, cross-check, quality gate, override condition), My Quality Standards, My Override Conditions, and The Benchmark Session. 2. Routing annotations — three clearly labelled blocks (ADD TO WORLDVIEW.MD / ADD TO BRAND.MD / ADD TO EXECUTION.MD). Each block contains 3-4 routing instructions specific to that file — when the AI should load it, what triggers it, what it checks against before handing off. Present each in a code block. Be specific — reference their actual answers throughout.
